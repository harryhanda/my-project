PROJECT SUMMARY: Automated CI/CD Pipeline for a Dockerized React App
===================================================================

--------------------
1. TECHNOLOGIES USED
--------------------
- React.js: Frontend library for the web application.
- Node.js & npm: Runtime environment for testing (npm ci) and building (npm run build).
- Docker: Used to containerize the application with an efficient multi-stage build.
- Nginx: A lightweight web server to serve the static React files in the final container.
- GitHub Actions: The CI/CD platform used to automate the build, test, and deployment workflow.
- Docker Hub: The container registry for storing and distributing the final production-ready Docker image.
- YAML: The language used to define the GitHub Actions workflow steps in the build.yml file.

-----------------
2. PROJECT OVERVIEW
-----------------
This project outlines a complete CI/CD (Continuous Integration / Continuous Deployment) pipeline for a React application, fully automated using GitHub Actions and Docker.

It serves as a practical demonstration of automating the process from a code push to a deployable cloud artifact. When code is pushed to the 'react-project/Experiment-7/' repository path, the pipeline automatically:
1. Installs dependencies securely.
2. Builds the React application.
3. Packages the app into a production-ready Docker image.
4. Pushes the new image to the Docker Hub registry.

The primary objectives are to apply modern DevOps principles, implement automated workflows, create optimized container images using multi-stage builds, and manage credentials securely via GitHub Secrets.

-------------------
4. CI/CD WORKFLOW
-------------------
The application's build and deployment are controlled by the GitHub Actions workflow defined in '.github/workflows/build.yml'.

i) Multi-Stage Dockerfile (react-project/Experiment-7/Dockerfile)
This file defines the blueprint for the production image. It uses two separate stages to create a final image that is small, secure, and fast, as it separates build-time dependencies from the runtime environment.

--- Dockerfile Snippet ---
# ---- Stage 1: Build the React app ----
FROM node:20 AS build
# Use a specific Node image for the build environment

WORKDIR /app
COPY package*.json ./

# Securely install exact dependencies
RUN npm ci --silent

COPY . .
# Build the static React app
RUN npm run build

# ---- Stage 2: Serve with nginx (production) ----
FROM nginx:1.25-alpine
# Use a lightweight Nginx image for the final container

# Copy only the built assets from the 'build' stage
COPY --from=build /app/dist /usr/share/nginx/html

EXPOSE 80
CMD ["nginx", "-g", "daemon off;"]
--- End Snippet ---


ii) Workflow Triggers and Defaults (from .github/workflows/build.yml)
This section defines that the workflow should only run for changes inside the 'Experiment-7' folder. It also sets the 'working-directory' for all subsequent commands, which is critical for operating within a monorepo.

--- build.yml Snippet (Triggers) ---
on:
  push:
    branches: [ main ]
    paths:
      - 'react-project/Experiment-7/**'
  workflow_dispatch: # Allows manual run

jobs:
  build:
    runs-on: ubuntu-latest
    defaults:
      run:
        # All commands will run inside this specific folder
        working-directory: react-project/Experiment-7
--- End Snippet ---


iii) Automated Build & Test (from .github/workflows/build.yml)
This step checks out the code, sets up the Node.js environment, and runs 'npm ci'. Using 'npm ci' ensures the build is reproducible by adhering strictly to the 'package-lock.json' file.

--- build.yml Snippet (Build/Test) ---
steps:
- name: Checkout Repository
  uses: actions/checkout@v4

- name: Set up Node.js
  uses: actions/setup-node@v4
  with:
    node-version: '20'

- name: Install dependencies and build
  run: |
    npm ci
    npm run build
--- End Snippet ---


iv) Docker Build & Push to Registry (from .github/workflows/build.yml)
This is the "Continuous Deployment" part. The workflow securely logs into Docker Hub using credentials stored in GitHub Secrets. It then builds the image from the Dockerfile, tags it, and pushes it to the Docker Hub registry.

--- build.yml Snippet (Push) ---
- name: Login to Docker Hub
  if: github.ref == 'refs/heads/main'
  uses: docker/login-action@v3
  with:
    username: ${{ secrets.DOCKERHUB_USERNAME }}
    password: ${{ secrets.DOCKERHUB_TOKEN }}

- name: Push Image to Docker Hub
  if: github.ref == 'refs/heads/main'
  run: |
    docker tag react-experiment-7 ${{ secrets.DOCKDOCKERHUB_USERNAME }}/react-experiment-7:latest
    docker push ${{ secrets.DOCKERHUB_USERNAME }}/react-experiment-7:latest
--- End Snippet ---
